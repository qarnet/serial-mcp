# Serial MCP Work Plans

Plans split into focused units so design and implementation can happen one step at a time.

- `PLAN-1-subscribe.md` - build RX session core, then migrate RX tools in phases
- `PLAN-2-result-metadata.md` - add structured stop/truncation metadata
- `PLAN-3-buffer-budgets.md` - add global/program and per-tool buffer budgeting
- `PLAN-4-no-new-rx-timeout.md` - add silence-based stop conditions
- `PLAN-5-event-context.md` - add event-context return shaping for pattern tools

Backlog-only ideas stay in `FEATURES.md` and are not part of these execution plans.
